# 给拥有服务器的主人们的部署指南 (Backend Docs)

这个文档是写给持有自己云端基础设施的开发者的。你需要跑起这个后端服务，以此支撑所有拿你 js 脚本白嫖的小白们的 OCR 运算开销。

## 1. 安装核心依赖
把你项目里的 `backend` 文件夹打包传到你的物理机或云主机的任意路径下，请确认你装了 `Python 3.8` 以上的版本。
进去目录，跑一句：
```bash
pip install -r requirements.txt
```

## 2. 发起守护节点
`ddddocr` 的极度轻量赋予了他在极烂配置主机上亦可健跑的能力。
最简单的挂载模式：
```bash
python main.py
```
这会占用前台终端。为了让所有终端都断开时服务仍能支撑你的亲友们使用，你可以走 `nohup`：
```bash
nohup uvicorn main:app --host 0.0.0.0 --port 8000 &
```

## 3. 把引擎大炮准星交给前端 (必须做！)
去你刚才保存的 `frontend/auto_captcha.user.js` 文件的第一部分，找到这段变量声明：
```javascript
let OCR_SERVER_URL = GM_getValue("ocr_server_host", "http://127.0.0.1:8000");
```
将里面的 `"http://127.0.0.1:8000"` 换成你刚才部署服务端的**真实公网IP + 端口 (或者域名)**！
*注意格式，结尾不要带斜杠 `/`。*

现在把你改好保存的 `auto_captcha.user.js` 发给那些还在用肉眼对焦验证码的小白朋友们吧！
